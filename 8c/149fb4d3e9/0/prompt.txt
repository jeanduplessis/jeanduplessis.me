## Steps

### Step 1: Update `package.json` version ranges
- Update all dependency and devDependency version ranges to target versions listed above
- File: `package.json`

### Step 2: Convert ESLint to flat config (required by ESLint 10)
ESLint 10 removes the legacy `.eslintrc.*` config format. Must migrate to flat config.

- **Delete** `.eslintrc.json`
- **Create** `eslint.config.mjs` with flat config format using `eslint-config-next`:
  ```js
  import { FlatCompat } from "@eslint/eslintrc";

  const compat = new FlatCompat();

  export default [...compat.extends("next/core-web-vitals")];
  ```
  Note: May need to install `@eslint/eslintrc` as a devDependency if `eslint-config-next` v16 doesn't natively export flat config. Will verify during implementation.

### Step 3: Update lint script in `package.json` (required by Next.js 16)
Next.js 16 removes the `next lint` command.

- Change `"lint": "next lint"` → `"lint": "eslint ."`
- File: `package.json`

### Step 4: Convert Stylelint config to ESM (required by Stylelint 17)
Stylelint 17 removes CommonJS support for the Node.js API.

- **Rename** `stylelint.config.js` → `stylelint.config.mjs`
- Convert `module.exports = { ... }` → `export default { ... }`
- Convert `plugins: ["stylelint-order", "stylelint-prettier"]` to ESM string references (these should still work as strings)
- File: `stylelint.config.mjs`

### Step 5: Run `npm install`
- Delete `node_modules` and `package-lock.json` for a clean install
- Run `npm install` to install all updated dependencies
- Resolve any peer dependency conflicts

### Step 6: Verify the build
- Run `npm run build` to confirm the project builds successfully
- Run `npm run lint` to confirm linting works with the new config
- Fix any build or lint errors that arise
